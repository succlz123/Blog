什么是启动框架

把 App 启动时候 Application 那一堆东西按照 Task 分类，根据依赖关系和优先级排序  (又比如 Gradle 中 Task 的前后依赖也差不多)

目的

  1. 很多 Task 其实可以延后执行的，只执行必要的任务。
  2. 按照 Task 组织后也方便进行测量耗时，进而优化。

    收益

加快启动速度，对于 Android 应用来说这种框架原理很简单，但是却是必须的。

拓扑排序

启动框架的设计, 对于 注解 和接口怎么设计都是比较无所谓, 不管是用 APT 还是插入字节码 Asm 实现都可以, 最核心的就是 Task 的前后依赖该怎么排序, 这就涉及到这个拓扑排序了, 它不是一种排序算法, 而是一种让有向图的顶点生成一个线性序列的算法。
Wiki 百科上：

对一个有向无环图(Directed Acyclic Graph 简称 DAG)G 进行拓扑排序，是将 G 中所有顶点排成一个线性序列，使得图中任意一对顶点 u 和 v，若边<u,v>∈E(G)，则 u 在线性序列中出现在 v 之前。通常，这样的线性序列称为满足拓扑次序(Topological Order)的序列，简称拓扑序列。简单的说，由某个集合上的一个偏序得到该集合上的一个全序，这个操作称之为拓扑排序。

DGA 有向无环图

  • AOV 网：数据在顶点 可以理解为面向对象
  • AOE 网：数据在边上，可以理解为面向过程！

特点

  • 图中每个顶点只出现一次。
  • A 在 B 前面，则不存在 B 在 A 前面的路径。(不能成环``！！！！)
  • 顶点的顺序是保证所有指向它的下个节点在被指节点前面！(例如 A—>B—>C 那么 A 一定在 B 前面，B 一定在 C 前面)。这个核心规则下只要满足即可，所以拓扑排序**序列不一定唯一**！

伪代码

  • 寻找一个入度为 0 的顶点，该顶点是拓扑排序中的第一个顶点序列，将之标记删除，
  • 然后将与该顶点相邻接的顶点的入度减 1
  • 再继续寻找入度为 0 的顶点
  • 直至所有的顶点都已经标记删除或者图中有环

关键是先寻找入度为 0 的顶点

方式 1 - 每次遍历法

  1. 遍历整个图中的顶点，找出入度为 0 的顶点，然后标记删除该顶点，更新相关顶点的入度
  2. 由于图中有 V 个顶点，每次找出入度为 0 的顶点后会更新相关顶点的入度，因此下一次又要重新扫描图中所有的顶点。

故时间复杂度为 O(V^2)

方式 2

由于删除入度为 0 的顶点时，只会更新与它邻接的顶点的入度，即只会影响与之邻接的顶点。
但是上面的方式却遍历了图中所有的顶点的入度。
改进

  1. 先将入度为 0 的顶点放在栈或者队列中。 - beforeCount == 0
  2. pop - 删除一个顶点 v，然后更新与顶点 v 邻接的顶点的入度。node.next.foreach { it.beforeCount– }
  3. 只要有顶点的入度降为 0，则将之入队列。 - beforeCount == 0 -> stack.push
  4. 此时拓扑排序就是顶点出队的顺序。用 list 记录下出队的顺序

该算法的时间复杂度为 O（V+E）

拓扑排序方法的实现

类似于二叉树的层序遍历, 其实说起来就广度优先遍历。

  1. 遍历图中所有的顶点，将入度为 0 的顶点 入队列。
  2. 从队列中出一个顶点，打印顶点，更新该顶点的邻接点的入度(减 1)，如果邻接点的入度减 1 之后变成了 0，则将该邻接点入队列。
  3. 一直执行上面 第二步，直到队列为空。
